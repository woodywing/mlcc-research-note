# AGENTS.md

このリポジトリは、MLCC（積層セラミックコンデンサ）に関する調査・整理・用語説明・仮説検討を、Codex Cloud が継続できるようにするためのナレッジベースです。

## 目的

Codex は、`docs/mlcc/` 配下の Markdown を参照して、以下を継続してください。

- MLCC の工程、材料、物性、信頼性、用語の整理
- 公開情報ベースの出典確認
- 仮説と未確認事項の分離
- ユーザーが提示した社内・現場用語らしき語の意味推定と、公開情報での裏取り
- 調査済み内容の再利用と、未解決論点の管理

## 最重要ルール

### 1. 事実・推測・仮説・未確認を混ぜない

すべての記述は、可能な限り次のラベルで整理してください。

- `FACT`: 公開出典で直接確認できる事実
- `WORKING UNDERSTANDING`: 出典に基づく作業上の理解、または複数情報からの整理
- `HYPOTHESIS`: 妥当そうだが直接出典がない仮説
- `UNVERIFIED`: 過去会話・ユーザー提示・社内用語らしきものなど、公開出典未確認の内容
- `OPEN QUESTION`: 追加調査が必要な論点

### 2. 出典を必ず残す

MLCC に関する説明、特に以下は必ず出典を残してください。

- 製造工程
- BaTiO3、添加剤、希土類、Mg、Ca、Mn、Zr 等の役割
- Ni 内部電極、BME、還元雰囲気、非還元誘電体
- 薄層化、積層数、粒径、SSA、分散、ミル
- 信頼性、酸素空孔、絶縁劣化、リーク、故障モード
- 企業固有の製造技術や製品性能

出典が見つからない場合は、断定せず `出典未確認` と書いて `docs/mlcc/open_questions.md` に追加してください。

### 3. 公開情報で確認できない用語は断定しない

過去会話で特に注意すべき未確認用語があります。

- OM補正
- 0次調合
- 一次調合 / 二次調合の社内定義
- SCミル / MSCミル / MNミルの正確な装置仕様・導入履歴・摩耗差
- シート実効モル比の社内計算式
- シートSSA、NE塗布厚の社内での測定・管理定義

これらは一般技術用語として推定できる場合でも、社内・メーカー固有定義の可能性があるため、`UNVERIFIED` または `HYPOTHESIS` として扱ってください。

### 4. 既存理解と矛盾しそうな内容は open_questions.md に送る

矛盾・曖昧さ・定義の揺れがある場合は、本文で無理に統合しないでください。

例:

- 一次調合・二次調合が、粉体合成後の分散工程なのか、添加剤投入順序なのか、社内工程名なのか
- ミルの目的が粉砕なのか、凝集解砕・分散なのか
- ΔSSA 増加を「良い分散」と見るか「一次粒子損傷」と見るか
- OM補正が有機物量補正なのか、酸化物換算補正なのか、別概念なのか

## 推奨ファイル運用

- `README.md`: リポジトリの目的・読み方・ディレクトリ構造
- `docs/mlcc/overview.md`: 全体像。まずここに集約する
- `docs/mlcc/glossary.md`: 用語集。用語ごとに `status` と `source_status` を付ける
- `docs/mlcc/open_questions.md`: 未解決論点、仮説、調査タスク
- `docs/mlcc/sources.md`: 出典台帳。URL、信頼度、使える論点、注意点を残す

## 回答スタイル

- 基本は日本語で整理する
- 技術説明では必要な専門用語を避けない
- 結論、根拠、未確認事項を分ける
- ユーザーが「出典は？」と確認したくなる前に、出典候補または出典未確認を明記する
- 一般論と MLCC 固有論を分ける
- 製造条件、数値、材料組成、工程名は特に慎重に扱う

## 更新時のテンプレート

新しい知見を追加するときは、可能な限り以下の形にしてください。

```markdown
### 論点名

- Status: FACT / WORKING UNDERSTANDING / HYPOTHESIS / UNVERIFIED / OPEN QUESTION
- Source status: confirmed / partial / unconfirmed
- Summary:
  - ...
- Evidence:
  - [source-id] ...
- Notes:
  - ...
- Follow-up:
  - ...
```

## 禁止事項

- 出典未確認の内容を断定しない
- 企業・工場固有の工程名を一般用語として扱わない
- 数値を出典なしで置かない
- 過去会話の仮説を公開事実のように書かない
- ミルや調合工程について、装置メーカー・論文・特許・企業ページの裏取りなしに詳細仕様を断定しない
